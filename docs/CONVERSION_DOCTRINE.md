# Hardware-Splicer — Strong conversion doctrine

**Status:** Supersedes the timid “5 DMs” framing in spirit. Keep [`CONVERSION_PUSH.md`](CONVERSION_PUSH.md) for funnel mechanics; **this doc is the strategy**.  
**Date:** 2026-07-18

## Honest self-critique

The earlier push (5 warm messages → 2 demos → 1 Sprint) is **survivable** when the founder is busy and the repo is private. It is **not** the best use of the research.

Research already said:

1. **Private flagship caps valuation** — discovery, grants, design partners, and inbound all die.  
2. **Best first dollar is B2B pilot / services**, not SaaS — but pilots need **credibility surface**, not only a zip in email.  
3. **Category is crowded on “KiCad MCP / AI PCB”** — weak play. **Salvage → carrier → gates → PROJECT_PACKAGE** is the scarce job.  
4. **Taiwan / cloud ecosystem programs** are real distribution + non-dilutive capital — ignored by a DM-only plan.  
5. **Compose Architon-class verify** raises price and trust — unused in the weak plan.  
6. You asked for **max asset valuation**, not “smallest outreach experiment.”

So: **B proof pack is the kit. A public open-core is conversion infrastructure. Design partners + ecosystem + PLG beat spray DMs.**

---

## Doctrine (what “best” means here)

```text
1. Make the product findable and defensible (A with honesty)
2. Land 3 named design partners (paid or case-rights Sprint)
3. Insert into ecosystems that already buy hardware AI / Taiwan soft-landing
4. Product-led inbound via agent install (MCP + quickstart) — not Product Hunt
5. Only then scale Sprints / site license
```

Warm DMs remain a **tactic inside (2)**, not the whole strategy.

## Strong plan — three engines in parallel

### Engine I — Credibility surface (do first, fast)

**Why research says so:** Invisible private repo = underpriced asset. Architon, KiCad MCP leaders, and grant reviewers only price what they can open.

| Move | Detail |
|------|--------|
| **Accelerate A** | Don’t wait 14 days of silence. Ship **LICENSE + public repo + alpha.16 prerelease + proof pack as release assets** as soon as pack exists (Day 0–3 of push). Keep B only for the hours until assets are ready. |
| **Public case artifacts** | Install reports + sample sprint zip + cold-exit honesty on the release — not buried in private chat |
| **Positioning line** | “Bring-up diligence for donor/salvage hardware” — explicitly **not** chat-to-PCB, **not** Architon clone; optional Architon after compose |

**Success:** Stranger can find repo, install story, and offer without a personal intro.

---

### Engine II — Design-partner cohort (conversion core)

**Not** “message 5 friends.” **Recruit 3 design partners** with a written deal.

| Element | Spec |
|---------|------|
| **ICP** | (1) repair/robot café or salvage lab, (2) 2–15 person prototype/EMS-adjacent team, (3) uni embedded lab with real boards |
| **Offer** | Discounted or full **Splice Sprint** in exchange for: redacted case note + quote permission + one intro |
| **Quota** | **3 partners in 60 days** — stop at 3, deliver deep |
| **Bar** | They must have a **real next board/donor** within 30 days (not “curious about AI”) |
| **Pricing** | Pilot band: lower half of NT$15k–60k, or case-rights trade for partner #1 only |

**How to find them (stronger than random DM):**

1. **Named list of 15** (founder + agent research): TW repair cafés, robotics clubs, small EMS, YZU-adjacent labs, Discord/Telegram hardware rooms you already touch  
2. **Outbound with public proof** (after A): “Here’s the prerelease + cold-exit — looking for 3 design partners this quarter”  
3. **Inbound CTA** on README: Design partner application (email + “what board / when”)  
4. **One vertical wedge:** lead with **robot / RC / repair-café** — you already have golden case + comparison doc  

**Success:** 3 signed partner slots or 3 scheduled intakes with real hardware dates.
